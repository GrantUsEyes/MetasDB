MetasDB FAQ page

What is "the simulation" and how does it work?


The simulation is a simulation built by Zackary Wright in Netlogo which features an organ represented by a mass of green dots on the grid (called patches) and one tumor cell represented by one red dot on the grid (called a "turtle". The simulation allows the user to initialize how high the tumor divide, metastasis and seeding probilities are along with whether or not CAFs are to be included in the simulation via sliders. The sliders start at a 0 initial chance and can go up to 1. Because cancer is more likely to spread over time naturally the simulation will automatically increase these probabilities over time.(see figure 1)

How do you start the simulation?


After the user initializes their variables, the user simply presses "setup" and "go". The simulation then graphs and keeps count of how many CAFs and tumor cells are on in the simulation as it runs. You can automate this process by clicking tools, then behavior space in the dropdown menu where you can then customize your run and have the simulation try different values of each slider as it goes.(see figure 2)



Why are the .csv files output by the simulation so large?


In order to have a more robust dataset and therefore more accurate data, the behaviorspace model is set so that each slider controlled variable is initialized to only .1 chance and increments .25 every run until it hits 1. The simulation then does this with CAFs present and absent, while keeping track of the count of tumor cells and CAFs present for each. (see figure 3)



How do I interpret these results?


The results given on the MetasDB page are averages for comparision purposes between the overall time (measured in steps) it takes for the original tumor to grow and metastasize between the simulation overall, the simulation with CAFs present andthe simulation with CAFs absent, along with the average number of CAFs present in the simulation when the CAFs are set to on. It is important to note for the purposes of this simulation, the MAX number of timesteps in each run can only go up to 2500 steps. At 2500 steps the run is terminated and moves on to the next run. Because of this, at 2500 steps the "organism" in the simulation can essentially be considered deceased due to other causes rather than the metastasis of tumor measured in the simulation.(bottom left figure 3)


Other things to note:


It is important to note for the purposes of this simulation, the MAX number of timesteps in each run can only go up to 2500 steps. At 2500 steps the run is terminated and moves on to the next run. Because of this, at 2500 steps the "organism" in the simulation can essentially be considered deceased due to other causes rather than the metastasis of tumor measured in the simulation. This is to prevent the simulation from supplying too much data or taking far to long to metastasize, as running a Behaviorspace module can already take quite a bit of time.


This simulation is not all encompassing! The simulation only measures and takes into account the interactions between CAFs and tumor cells and postulate how the CAFs may modify a tumors probability to divide, metastasize and seed. It does not take into account other interactions going on, within and on the outside of, the organ cell. The user's modification of the probability sliders is essentially modifiying the "organism's" enviornment.
